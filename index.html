第一个烂尾项目

Apr 15th, 2014 | Comments

信心满满地准备修出一排GitHub项目，算是对自己有个交代，也能造福后代，比如Kindle推送、拯救Kindle、监控推送什么的还算是自己用着别人也可能有用的东西，见缝插针地重构着代码，希望能够早日上线。在我长长的ToDo List里舍不得删除任何一个项目，每一个都是曾今为之兴奋的想法，但面对现实，有限的时间和效益，那些在代码重构中予己无益、于人无用的项目还是放弃为好，比如typespeed背单词生成器。当年背着GRE单词并练习着盲打以提高写作打字速度，无聊中发现了这个typespeed可以熟悉键位，就把《要你命3000>结合艾宾浩斯曲线生成了单词循环的列表，替代typespeed自带的测试词表，这样就能边打字边背单词了= =我是有多蛋疼。。现在觉得熟悉键盘以后的练习打字活动就属于无谓的劳损了，而且背单词本身的效果也有争议，不做也罢，放上GitHub必须重构下否则不堪入目，但这种简单的项目只有体力活了，而且连我自己都不用了，这么小众而蛋碎的需求估计也没啥人会有了，就算有这种需求的人应该自己也会实现，所以就让它随风飘荡吧。瞬间轻松了。

迁移测试

Apr 5th, 2014 | Comments

迁移成功！

Dropbox + GitHub 真是好用哭了，开发配置都云端化，迁移环境的成本也越来越低，就像从Windows 8开始系统都和Hotmail账号绑定，如此一来重装系统的配置、App都是从云端同步的，从用户角度来说的确方便不少。对用户来说，这是云。其本质就是用户所有的资源都掌握在里企业的手里，以前用户的文件、配置都需要自己存储、操作、记忆，现在都集体托管在企业端再次通过集约化降低个人使用电脑的成本，从经济角度来说这是好的。但从反垄断和用户权益角度其实也是需要思考的，就像财阀掌握了经济政治的话语权，而企业掌握用户数据就像了解了用户的生活一切，怎么促销、分析、利用就看企业心情了，用户似乎真的变成了肉板上的大餐，任企业宰割。又想用集约化服务，又希望去中心化，哪里有这么好的事？或许这就是互联网的下一个形态，就像资本主义推翻封建集权主义一样。企业再次回到了通过硬件收费，软件免费的制度，谁知道呢，且行且探索。

Mar 26th, 2014 | Comments

自从用上Python，感觉生活一下子美好了不少，分分钟你的想法就可以实现，而且是最实用的那些，“刷个网页抢个票，下个小片摇个号”，甚至还因为Python和扯淡的研究方向找到了份装逼的实习。Python的好处在于语法的精炼、脚本的便捷、开源库的可扩展性，虽然编译速度和作为大型开发项目的能力欠缺，但是作为家庭小助手那是最适合不过的。这也就是为什么会有这么多编程语言存在的原因，满足不同的需求，每个人对不同任务的学习曲线、规模大小、适用范围、自由程度、敏捷程度、运行速度等需求都不尽相同，因此诞生了满足不同需求的各种工具，各取所需即可，没什么好争的。
